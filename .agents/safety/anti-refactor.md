# Anti-Refactor Safety

Hard rules:

- Không refactor unrelated files.
- Không rename public exports nếu chưa cần.
- Không đổi folder structure lớn nếu chưa được yêu cầu.
- Không thay thế pattern đang dùng ổn định bằng pattern mới chỉ vì sở thích.
- Không xóa code nếu chưa chắc không còn dùng.

Nếu thấy code cũ chưa tối ưu nhưng không ảnh hưởng task, ghi nhận thay vì sửa ngay.
