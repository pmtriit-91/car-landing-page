# Backward Compatibility

Mọi thay đổi phải ưu tiên tương thích ngược.

Kiểm tra:

- Component props cũ còn hoạt động không?
- API consumer cũ có bị lỗi không?
- State/localStorage/cache cũ có bị invalid không?
- UI flow cũ có bị mất bước không?

Nếu không thể tương thích ngược, phải báo rõ rủi ro.
