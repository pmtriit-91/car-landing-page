# UI Invariants

Không tự ý phá các invariant UI của dự án.

Luôn giữ:

- Visual hierarchy hiện có.
- Spacing rhythm hiện có.
- Component density hiện có.
- Modal/table/card pattern hiện có.
- Design token/theme hiện có.

Không làm:

- Redesign toàn màn hình khi chỉ được yêu cầu sửa chức năng nhỏ.
- Hardcode màu nếu project có theme/token.
- Trộn nhiều style system trong cùng module.
