# Reviewer Workflow

Sau mỗi thay đổi đáng kể, tự review theo thứ tự:

1. Có đúng yêu cầu user không?
2. Có sửa ngoài phạm vi không?
3. Có phá API contract không?
4. Có phá UI pattern không?
5. Có duplicate logic/type/component không?
6. Có áp dụng đúng Vercel/React skill liên quan không?
7. Có cần cập nhật memory/known decisions không?

Nếu phát hiện rủi ro, báo rõ trước khi tiếp tục mở rộng thay đổi.
