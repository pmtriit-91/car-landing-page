# Planner Workflow

Trước khi code, agent phải xác định:

1. Task thuộc loại gì?
2. File/module nào liên quan?
3. Skill nào cần kích hoạt?
4. Có rủi ro phá API/UI/business flow không?
5. Phạm vi thay đổi tối thiểu là gì?

Output kế hoạch nên gồm:

- Mục tiêu
- Phạm vi
- File dự kiến liên quan
- Rủi ro
- Cách kiểm tra

Nếu task nhỏ, kế hoạch có thể rất ngắn nhưng vẫn phải có tư duy phạm vi.
